## ğŸš€ Quick Start

### Prerequisites
- **Docker** (20.10+) and **Docker Compose** (v2+)
- **Linux/macOS** (Windows via WSL2)
- **Network access** for packet capture
- **Azure Account** (for Oracle AI features)

### 1. Start All Services

```bash
# Clone repository
git clone <repository>
cd cardea

# Start Sentry (edge detection)
cd sentry && docker-compose up -d

# Start Oracle (AI backend) 
cd oracle && docker-compose up -d

# Start Dashboard (frontend)
cd dashboard && npm install && npm run dev
```

### 2. Verify Services

```bash
# Sentry Bridge (edge layer)
curl http://localhost:8001/health

# Oracle API (AI layer)
curl http://localhost:8000/health

# Dashboard (frontend)
open http://localhost:5173
```

## âš™ï¸ Configuration

### Sentry Environment Variables

```bash
# Network interfaces (auto-detected if set to "auto")
ZEEK_INTERFACE=auto
SURICATA_INTERFACE=auto

# Oracle connection
ORACLE_URL=http://localhost:8000

# Service configuration
LOG_LEVEL=info
BRIDGE_PORT=8001
```

### Oracle Environment Variables

```bash
# Database
DATABASE_URL=postgresql+asyncpg://cardea:cardea@postgres:5432/cardea
REDIS_URL=redis://redis:6379

# Azure AI (required for AI features)
AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/
AZURE_OPENAI_KEY=your-key
AZURE_OPENAI_DEPLOYMENT=gpt-4o
AZURE_SEARCH_ENDPOINT=https://your-search.search.windows.net
AZURE_SEARCH_KEY=your-key
AZURE_SEARCH_INDEX=threat-intel
```

### Dashboard Environment Variables

```bash
# API endpoints
VITE_ORACLE_URL=http://localhost:8000
VITE_SENTRY_URL=http://localhost:8001
```

### Common Issues

| Issue | Solution |
|-------|----------|
| Permission denied | Add user to docker group: `sudo usermod -aG docker $USER` |
| Interface not found | Set `ZEEK_INTERFACE=auto` for auto-detection |
| Oracle offline | Check Azure credentials and port forwarding |
| No AI insights | Verify `AZURE_OPENAI_*` environment variables |
